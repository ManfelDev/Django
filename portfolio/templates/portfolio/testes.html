Test Types

This application uses three main types of tests to ensure quality and stability:

  • Unit Tests: Validate individual methods or model behavior in isolation.
  • Integration Tests: Verify how different components (models, views, URLs) work together.
  • Template Tests: Ensure correct rendering of HTML pages and expected content.

Integration Tests

Integration tests were written to confirm that the application's routing and views return the correct responses and templates. For example:

  • Views respond with status 200 and use the correct HTML templates.
  • URLs resolve to the expected view functions.
  • Models are loaded properly from the database fixture and have correct relationships.

All tests were executed using Django's test runner.

Example Test

def test_tecnologias_view(self):
response = self.client.get(reverse('portfolio:tecnologias'))
self.assertEqual(response.status_code, 200)
self.assertTemplateUsed(response, 'portfolio/tecnologias.html')

This test checks that the page for viewing technologies is available and rendered with the correct template.

Selenium Functional Tests

To validate the behavior of the application in a real browser environment, Selenium was used to simulate user interaction across different features of the site. Below are three recorded demonstrations:

1. Navigation Test

Test Steps:

  • Visit the homepage.
  • Navigate through several public pages.
  • Validate specific titles and content on each page.

2. Authentication Test

Test Steps:

  • Register a new user.
  • Login with the registered credentials.
  • Create and publish a new article.
  • Delete the created article.
  • Logout the user.

3. Project Lifecycle Test

Test Steps:

  • Register a new user.
  • Login with the registered credentials.
  • Create a new course project.
  • Edit the project's information.
  • Delete the created project.
  • Logout the user.

These tests demonstrate how Selenium automates browser actions to ensure the main features of the application behave correctly under real usage scenarios.

Coverage Report

The tests achieved an overall coverage of 75%. Key modules such as portfolio.models, portfolio.forms, and portfolio.tests reached full coverage, while the view logic had lower coverage due to its complexity and interaction with the database and templates.
