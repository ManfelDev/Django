Technical Presentation

This page presents the technical details of the Django web application that powers this portfolio site, covering modern web development concepts and deployment strategies.

Main Technologies Used

  • Backend: Django 5 + Django Ninja
  • Frontend: HTML5, CSS3, Django Templates
  • Database: MySQL
  • Version Control: Git + GitHub
  • Deployment: PythonAnywhere

Application Architecture (MVT)

  • Model: Defines the structure of the database, including fields and relationships.
  • View: Handles HTTP requests, processes data via models, and passes context to templates.
  • Template: Generates the HTML presentation layer using the context provided by views.

API RESTful Implementation with Django Ninja

This application includes a comprehensive RESTful API that enables external systems or developers to interact programmatically with the data stored in the site. The API follows REST (Representational State Transfer) principles and is implemented using Django Ninja, a lightweight framework built on top of Django and Pydantic.

  • RESTful Design: All endpoints follow RESTful patterns like /api/bands (GET, POST), /api/bands/{id}/ (GET, PUT, DELETE), ensuring consistency and predictability.
  • HTTP Methods: Proper use of GET (retrieve), POST (create), PUT (update), DELETE (remove) operations.
  • Status Codes: Returns appropriate HTTP status codes (200, 201, 404, 500) for different scenarios.
  • JSON Communication: Uses JSON format for data exchange, making it language-agnostic.
  • Resource-Based URLs: URLs represent resources rather than actions, following REST conventions.

Django Ninja Framework

Django Ninja is a modern, fast web framework for building APIs with Django, based on standard Python type hints. It provides several advantages:

  • Type Safety: Uses Pydantic schemas for automatic request/response validation and serialization.
  • Auto Documentation: Automatically generates interactive API documentation using Swagger/OpenAPI.
  • Performance: Built for speed with minimal overhead compared to Django REST Framework.
  • Modern Python: Leverages Python 3.6+ type hints for better code quality and IDE support.
  • Easy Integration: Seamlessly integrates with existing Django projects and ORM.

CI/CD (Continuous Integration/Continuous Deployment)

CI/CD practices ensure reliable and automated software delivery. This project implements CI/CD concepts through:

  • Version Control: Git workflow with feature branches and pull requests for code review.
  • Automated Testing: Unit tests, integration tests, and Selenium functional tests run automatically.
  • Code Quality: Consistent coding standards and automated code analysis.
  • Deployment Pipeline: Streamlined deployment process from development to production.
  • Environment Management: Separate development, testing, and production environments.

GitHub Actions

GitHub Actions is a powerful CI/CD platform that automates workflows directly within GitHub repositories. The project utilizes GitHub Actions for:

  • Automated Testing: Runs test suite on every push and pull request to ensure code quality.
  • Code Validation: Checks code formatting, linting, and security vulnerabilities.
  • Build Automation: Automatically builds the application and dependencies.
  • Deployment Triggers: Automates deployment to production environments when changes are merged.
  • Notification System: Sends alerts about build failures or successful deployments.

Docker Containerization

Docker enables consistent application deployment across different environments. This project incorporates Docker concepts:

  • Containerization: Packages the application with all its dependencies into a portable container.
  • Environment Consistency: Ensures the application runs identically in development, testing, and production.
  • Scalability: Enables easy horizontal scaling by spinning up multiple container instances.
  • Isolation: Provides process and resource isolation for better security and stability.
  • Microservices Ready: Facilitates breaking down monolithic applications into smaller, manageable services.

Data Modeling (Entity-Relationship Diagram)

The data is organized using Django models. Below is a summary of the key entities and their relationships:

  • Project – Represents a student-developed game or experience. Each project includes:
    • A title and description
    • Optional links (Itch.io, GitHub, YouTube)
    • Technical aspects and core concepts
    • A ForeignKey to one Discipline
    • A ManyToMany relation with Technology
    • A OneToOne relation with TechnicalSheet
    • A OneToMany relation with ProjectImage
  • Discipline – Represents a course from the university. It has:
    • Name, year, semester, and optional Moodle/ULusófona page links
    • A ManyToMany relation with Professor
    • A OneToMany relation with Project
  • Technology – Each technology used (e.g., Unity, Blender). Includes:
    • Name, logo, link, and description
    • A ManyToMany relation with Project
  • Professor – Teachers associated with each discipline.
    • A ManyToMany relation with Discipline
  • ProjectImage – Images illustrating the project.
    • Each has a ForeignKey to one Project
  • TechnicalSheet – Holds extra metadata (platform, team size, dev duration).
    • Each is linked via a OneToOne to one Project
